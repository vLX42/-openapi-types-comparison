# Project Summary

## OpenAPI Code Generator Comparison Project

This project has been successfully transformed into a comprehensive comparison tool for evaluating `openapi-typescript` vs `@hey-api/openapi-ts`.

## 📁 Project Structure

```
.
├── swagger.json                      # Source OpenAPI spec (MVS Api 5.0)
├── package.json                      # NPM scripts and configuration
├── README.md                         # Main documentation
├── QUICKSTART.md                     # Quick start guide
├── .gitignore                        # Ignore generated outputs
│
├── compare-stats.mjs                 # Statistics comparison tool
├── compare-outputs.mjs               # Detailed code comparison tool
├── view-samples.mjs                  # Code sample viewer
├── generate-comparison-report.mjs    # Markdown report generator
│
├── output-openapi-typescript/        # Generated by openapi-typescript
│   └── types.ts                      # TypeScript types (51 KB, 1535 lines)
│
├── output-hey-api/                   # Generated by @hey-api/openapi-ts
│   ├── types.gen.ts                  # Type definitions
│   ├── sdk.gen.ts                    # SDK/service functions
│   ├── client/                       # Client implementation
│   └── core/                         # Core utilities
│
└── COMPARISON_REPORT.md              # Generated comparison report
```

## 🎯 Available Commands

| Command | Description |
|---------|-------------|
| `npm run gen:openapi-typescript` | Generate types using openapi-typescript |
| `npm run gen:hey-api` | Generate client using @hey-api/openapi-ts |
| `npm run gen:all` | Generate outputs from both tools |
| `npm run compare:stats` | Show statistics comparison |
| `npm run compare:samples` | Show code samples side-by-side |
| `npm run compare` | Show detailed code comparison |
| `npm run report` | Generate markdown comparison report |
| `npm run full-compare` | **Run everything: generate + compare + report** |

## 🚀 Quick Start

```bash
npm run full-compare
```

This single command will:
1. Generate code from both generators
2. Display statistics in terminal
3. Create a detailed markdown report

## 🎓 Key Insights
